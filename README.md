# Todo App Automation

End-to-end test automation suite for the [Playwright Practice Todo App](https://github.com/vibhorgoyal18/test-todo-app), built with **Playwright**, **TypeScript**, **playwright-bdd** (Gherkin/BDD), and **Allure** reporting.

The Allure report is automatically published to **GitHub Pages** on every push to `main`.

## Project Structure

```
todo-app-automation/
├── .github/
│   └── workflows/
│       └── playwright.yml       # GitHub Actions CI/CD pipeline
├── src/
│   ├── fixtures/
│   │   └── index.ts             # Custom Playwright-BDD fixture & step exports
│   ├── steps/
│   │   ├── auth.steps.ts        # Login / logout step definitions
│   │   ├── todos.steps.ts       # Todo CRUD & filter step definitions
│   │   ├── dashboard.steps.ts   # Dashboard step definitions
│   │   └── profile.steps.ts    # Profile management step definitions
│   └── utils/
│       ├── userData.ts          # Reads users.csv by user key
│       └── configReader.ts      # Reads config.properties for the active env
├── testData/
│   ├── local/
│   │   ├── users.csv            # Local test user credentials
│   │   └── config.properties    # Local environment config (baseURL etc.)
│   └── prod/
│       ├── users.csv            # Prod test user credentials
│       └── config.properties    # Prod environment config
├── tests/
│   ├── auth.feature             # Authentication scenarios
│   ├── todos.feature            # Todo management scenarios
│   ├── dashboard.feature        # Dashboard scenarios
│   └── profile.feature          # Profile management scenarios
├── playwright.config.ts
├── tsconfig.json
└── package.json
```

> `.features-gen/` is auto-generated by `bddgen` at runtime — do not edit manually.

## Test Data & Environments

Test data is stored under `testData/` and is environment-aware. The active environment is controlled by the `ENV` variable (defaults to `local`).

### `users.csv`

Credentials are never hardcoded in tests. Feature files reference users by **key**, and the step definitions resolve the full credentials from the CSV at runtime.

```csv
key,email,name,role,password
testuser,alice@test.com,Alice Tester,admin,Test@1234
viewer,bob@test.com,Bob Viewer,viewer,View@5678
```

**Usage in feature files:**
```gherkin
Given I am logged in as "testuser"
When I enter credentials for user "viewer"
```

### `config.properties`

Non-sensitive environment configuration:

```properties
base_url=http://localhost:5175/test-todo-app
app_name=Playwright Practice Todo App
default_browser=chromium
timeout=30000
```

### Switching Environments

```bash
# Run against local (default)
npm test

# Run against prod
ENV=prod npm test

# Override baseURL directly
BASE_URL=https://example.com npm test
```

## Running Tests

### Run all tests (headless)
```bash
npm test
```

### Run with browser visible
```bash
npm run test:headed
```

### Run with Playwright UI mode
```bash
npm run test:ui
```

### Run in debug mode
```bash
npm run test:debug
```

### Run a specific feature tag
```bash
npx bddgen && npx playwright test --grep @auth
npx bddgen && npx playwright test --grep @todos
npx bddgen && npx playwright test --grep @dashboard
npx bddgen && npx playwright test --grep @profile
```

### Run on a specific browser
```bash
npx bddgen && npx playwright test --project=chromium
npx bddgen && npx playwright test --project=firefox
npx bddgen && npx playwright test --project=webkit
```

> **Note:** `bddgen` must always run before `playwright test`. It compiles `.feature` files into spec files under `.features-gen/`. The `npm test` script handles this automatically.

## Allure Reports

### Generate and open locally

```bash
# Step 1 — Run tests (produces allure-results/)
npm test

# Step 2 — Generate the HTML report
npx allure generate allure-results --clean -o allure-report

# Step 3 — Open in browser
npx allure open allure-report
```

### Quick serve (generate + open in one command)
```bash
npx allure serve allure-results
```

### Install Allure CLI globally (optional, avoids npx overhead)
```bash
npm install -g allure-commandline
allure serve allure-results
```

## CI/CD Pipeline

The GitHub Actions workflow at [`.github/workflows/playwright.yml`](.github/workflows/playwright.yml) runs on every push and pull request to `main`.

### What the pipeline does

```
1. Checkout automation repo
2. Install Node.js 20 + npm dependencies
3. Install Chromium browser
4. Checkout & build the test-todo-app (serves it on localhost:5175)
5. Wait for the app to be ready
6. Pull Allure history from the gh-pages branch  ← preserves trend data
7. Copy history into allure-results/history/
8. Run bddgen to generate spec files
9. Run Playwright tests (Chromium)
10. Generate Allure HTML report (with history)
11. Upload report as a GitHub Actions artifact (30-day retention)
12. Deploy Allure report to GitHub Pages            ← only on main branch
```

### Triggering manually

In the GitHub repository, go to **Actions → Playwright BDD Tests → Run workflow**.

### Viewing the live Allure report

After the first successful pipeline run, the report is published at:

```
https://vibhorgoyal18.github.io/todo-app-automation/allure-report/
```

> **One-time setup:** Go to the repository **Settings → Pages** and set the source branch to `gh-pages`.

### Artifacts

Each pipeline run uploads two artifacts (retained for 30 days):
- `allure-report` — Full Allure HTML report
- `playwright-report` — Playwright's built-in HTML report

These can be downloaded from the **Actions** tab of any workflow run.
